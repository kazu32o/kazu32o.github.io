2. Javaの学習成果

  • Java でアプリケーションを開発するときには、IntelliJ、NetBeans や Eclipse などの※統合開発環境 (IDE) が利用される場合が多い。Javaコンパイラはjavac.exe
    ※統合開発環境(IDE=Integrated Development Environment)は開発用に色々なものが盛り込まれた(=エディタ、コンパイラ、実行ができる)ツール。（VScodeもそれ）
  • javaファイルをコンパイルすると、"*.class"ファイルが生成され、vscodeやcmd上で実行すると出力結果が表示される。（C言語でいう"*.out"ファイル）
  • オブジェクト指向はモノの作成と操作として見る考え方(4)（製造品でいえばガワ（寸法や色、材料など）と操作方法（機能）みたいに分けて考える。）
    ※300種類の車のプログラムを記述する場合、製造品のガワ・操作方法についてプログラムを一つにまとめると、コードが膨大で作成者以外にはわかりづらい。
    さらに、「やっぱり、車のアクセルのボタンは△じゃなくて〇でお願いします。」などと依頼されたとき、膨大なコードから300か所変更する必要がある。
    ここで必要なのがオブジェクト指向という考え方である。ガワ・操作方法で分けて記述しておくことで操作方法側のコードを書き換えるだけでいいし、コードを記述した作業者以外にも変更が容易になる。
